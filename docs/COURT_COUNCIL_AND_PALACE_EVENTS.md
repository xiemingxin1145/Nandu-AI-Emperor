# 大殿朝会与后苑内廷事件设计说明

本模块用于把宫殿待办从“任务卡”升级为“人物出场、互相上奏、皇帝裁断”的宫廷事件结构。

## 已接入

- `CourtCouncilSystem`
- `CouncilScene`
- `CouncilLine`
- `CouncilChoice`
- `PalaceTasksScreen` 的“入殿听奏”按钮

## 当前体验

玩家进入任意宫殿待办后，可以选择“入殿听奏”。系统会按宫殿类型生成一场轻量朝会：

1. 场景标题与摘要
2. 多名角色依次上奏
3. 角色之间形成支持、反对、忧虑、中立的态度差异
4. 皇帝看到多个裁断选项
5. 选择后自动生成圣旨草稿，进入朝议页

## 宫殿覆盖

- 垂拱殿：主战、主和、理财、武将同殿争论
- 枢密院：军报、守城、出击、粮道
- 政事堂：钱粮、市舶、漕运、民力
- 文德殿：召贤、授官、旧臣疑虑
- 御书房：密折、外交、真假缓急
- 皇城司：朝臣动向、耳目风险
- 后苑内廷：皇后、太后、内侍问对
- 太庙：正统、祭告、军心、礼制

## 设计原则

- 角色说话必须符合古代身份与南宋朝堂语境。
- 不使用现代网络词和现代管理话术。
- 每个事件至少给玩家两个以上裁断方向。
- 后宫不是单纯装饰，而是内廷稳定、皇帝声望、外朝观感的系统入口。
- 该模块借鉴橙光式宫廷剧情的“人物站位 + 对话 + 选择 + 后果预览”，但所有选择最终仍通过圣旨草稿和本地规则系统执行。

## 后续计划

- 把选择结果写入 GameState：忠诚、派系、声望、内廷稳定等。
- 给每个角色固定站位与立绘槽位。
- 让 AI 参与生成临场奏议，但仍由古代语气校验器过滤。
- 给后宫加入皇后、太后、内侍、宗室等长期关系线。
- 加入连续事件：拖延、重用、冷落、外朝风评。
