# 外交与外贸持久化骨架

本模块把西夏、大理、高丽、海贸诸商从“地图显示对象”推进为可变化、可保存的系统状态。

## 已接入

新增：

- `DiplomacyState`
- `TradeRouteState`
- `DiplomacyTradeSystem`

当前为了兼容旧存档，没有直接改 `GameState` 主结构，而是把状态编码进已经参与存档的 `Rumor` 列表中：

- `diplomacy_*`：外交状态
- `trade_*`：贸易路线状态

后续可以迁移成正式字段。

## 外交状态

每个外部势力包含：

- `powerId`
- `relation`：关系
- `pressure`：压力
- `trust`：互信
- `status`：状态描述
- `lastTouchedTurn`：最后变动回合

当前对象：

- 金国
- 西夏
- 大理
- 高丽
- 海贸诸商

## 外贸路线状态

每条贸易路线包含：

- `routeId`
- `isOpen`：是否开放
- `income`：收益
- `risk`：风险
- `control`：朝廷控制力
- `smuggling`：走私程度
- `lastTouchedTurn`：最后变动回合

当前路线：

- 泉州香料海贸
- 明州船材海路
- 广州南海商路
- 西夏马市
- 大理西南路

## 已接入到玩法

朝会选择现在会影响外交外贸状态：

- 政事堂“整顿市舶”会提升海贸关系、港市收益与控制力。
- 政事堂“安民轻敛”会降低部分南海私商风险。
- 御书房“遣使探问”会提升西夏、高丽关系与互信。
- 御书房“留中不发”会让部分外交互信略降。

## 待办派生

`WorldStrategySystem.diplomacyBriefs(state)` 会读取这些状态，动态生成：

- 外交待办
- 外贸待办
- 西夏马市待办
- 港市风险待办

## 后续计划

下一步应继续：

1. 把每旬贸易收入接入 `advanceTurn()`。
2. 在国政 / 政事堂页面展示贸易路线详情。
3. 在御书房展示外交关系详情。
4. 给贸易路线增加事件：海盗、走私、豪商干政、船队失期。
5. 给外交关系增加事件：使臣来朝、互市争议、边境误会、贡赐成本。

## 一句话

外交外贸不再只是地图上的颜色，而是会随皇帝裁断变化，并能被后续待办读取的长期系统。
